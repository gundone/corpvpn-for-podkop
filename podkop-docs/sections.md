# Секции -- Podkop

Источник: https://podkop.net/docs/sections/

## Введение
В **Podkop** можно направлять разный трафик в разные outbound (VLESS, WG, AWG, etc).

Пример: маршрутизация Geoblock списка на европейский VLESS сервер, Youtube на российский WG сервер, Google AI инструменты на USA Shadowsocks.

## OpenWrt LuCi Config Example

```
config section 'main'
    option connection_type 'proxy'
    option proxy_config_type 'urltest'
    list community_lists 'geoblock'

config section 'wg'
    option connection_type 'vpn'
    option interface 'wg0'
    list community_lists 'youtube'
```

Секций может быть любое количество.

## Создание и удаление секций

Секция по умолчанию: **Main** (создается при установке)

Правила именования: только латинские символы, цифры и символы подчеркивания

- Кнопка **Add**: создает новую секцию
- Кнопка **Delete**: находится справа от имени секции, удаляет секцию

## Настройки секций

### Тип подключения (Connection Type)

Четыре поддерживаемых типа: `Proxy`, `VPN`, `Block`, и `Exclusion`

### Тип Proxy

Поддерживаемые протоколы: `VLESS`, `Shadowsocks`, `Trojan`, `Hysteria2`, `Socks` (v4 и v5)

**Варианты типа конфигурации:**

**1. URL подключения (URL Connection)**
- Используется для одиночного подключения по конфигурационной ссылке

**2. Selector**
- Используется для работы с несколькими конфигурационными ссылками
- Позволяет вручную выбирать активное подключение из списка доступных

**3. URLTest**
- Используется для работы с несколькими конфигурационными ссылками с автоматическим выбором
- sing-box периодически проверяет каждое подключение, измеряя задержку
- Рекомендуется использовать для резервирования

**Дополнительные настройки URLTest:**
- **Интервал проверки URLTest** -- Интервал между проверками отклика серверов
- **Порог переключения URLTest** -- Значение задержки для принятия решения о переключении
- **URL для тестирования URLTest** -- Адрес для тестирования соединения

**4. Конфигурация Outbound**
- Позволяет внести Outbound, поддерживаемый sing-box, в формате JSON

**Дополнительная настройка:**
- **UDP over TCP** -- Передача UDP-пакетов в TCP соединении. Работает с Shadowsocks и Socks.

### Тип VPN

В этом режиме выбирается предварительно настроенный сетевой интерфейс.

Поддерживается: `Wireguard`, `AmneziaWG`, `OpenVPN`, `OpenConnect`, и любой другой туннель/VPN.

> Note: Достаточно настроить только сам туннель или VPN, настройки firewall и зон не обязательны

**Настройки:**
- **Сетевой интерфейс** -- Список доступных интерфейсов для выбора
- **Резолвер доменов (Domain Resolver)** -- Отдельный DNS-сервер для выбранного интерфейса. Все запросы, направляемые в этот интерфейс, будут разрешаться через выбранный DNS-сервер. DNS-запросы, отправляемые резолвером доменов, по умолчанию направляются в этот же туннель.

### Тип Block

Позволяет заблокировать определенные домены и/или подсети. Заблокированные элементы направляются в sing-box и отклоняются.

### Тип Exclusion

Позволяет исключить определенные домены и/или подсети из маршрутизации любой секции.

## Списки сообщества (Community Lists)

**Содержимое списка Russia inside:**
- GEO Block, Block, Porn, News, Anime, Youtube, Meta*, Twitter(X), HDRezka, Tik-Tok

**Дополнительные списки:**
- Meta*, Twitter, Discord, Telegram, Cloudflare, Google AI, Google Play, H.O.D.C.A., Hetzner ASN, OVH ASN, Digital Ocean ASN, CloudFront ASN

**Взаимоисключающие комбинации:** Russia inside, Russia outside, Ukraine -- не могут быть выбраны одновременно.

## Тип пользовательского списка доменов

Два режима:
- **Текстовый список** -- разделители: запятая, пробел или новая строка; поддержка комментариев `//`
- **Динамический список** -- добавление доменов по одному

Формат: `example.com`, `sub.example.com`

## Тип пользовательского списка подсетей

Два режима: Текстовый и Динамический (аналогично доменам)

Формат: `103.21.244.0/22`, `103.21.244.89`

## Локальные списки доменов/подсетей

Файлы вида `file.lst` или `file.txt`, каждый домен/подсеть с новой строки.

## Внешние списки доменов/подсетей (Remote Lists)

Загрузка с GitHub или личного сайта. Полный адрес с протоколом (`http://` или `https://`)

Поддерживаемые форматы: **.json | .srs | .lst | .txt**

## Полностью маршрутизируемые IP-адреса

Направляет весь трафик с конкретного устройства в туннель.

## Включить смешанный прокси (Enable Mixed Proxy)

Включает на роутере смешанный прокси-сервер (HTTP / SOCKS), использующий туннель данной секции.

Адрес прокси: IP адрес роутера + порт Mixed Proxy Port.

## Порт смешанного прокси (Mixed Proxy Port)

Задает порт, на котором будет доступен смешанный прокси-сервер (HTTP / SOCKS).
