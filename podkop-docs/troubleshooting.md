# Поиск и устранение неисправностей -- Podkop

Источник: https://podkop.net/docs/troubleshooting/

> **Самые частые проблемы:**
> - Не открываются заблокированные сайты на одном устройстве? Проверьте **настройки DNS на устройстве**.
> - Ошибка `FakeIP Status: ! does not work in browser`? Дело в **DNS на устройстве** или **«Безопасном DNS» в браузере**.

## Принципы работы маршрутизации

Маршрутизация по доменным именам работает при соблюдении условий:

1. **DNS-запросы** от клиентского устройства направляются на роутер с Podkop
2. **Роутер с Podkop** является **шлюзом** для всего трафика (или для подсети FakeIP `198.18.0.0/15`)
3. **dnsmasq** настроен на пересылку запросов на `127.0.0.42`
4. **sing-box** запущен, прослушивает `127.0.0.42:53`, конфигурация корректна
5. **Правила nftables** активны, маркируют и перенаправляют трафик через tproxy на `127.0.0.1:1602`
6. **Внешнее подключение** (прокси/VPN) работоспособно

## Ключевые выводы

- Если на одних устройствах работает, а на других нет -- проблема в **локальных настройках** (сторонние DNS или VPN-клиент)
- Диагностика корректна **только из локальной сети**

## Диагностика на клиентском устройстве

### Проверка FakeIP:
```
nslookup fakeip.podkop.fyi
```
Ожидаемый результат: IP из диапазона `198.18.x.x`, Server: `192.168.1.1`

### Проверка прохождения трафика через sing-box:
```
curl -v https://fakeip.podkop.fyi/check
```
Ожидаемый ответ: `{"fakeip": true, "IP": "XX.XX.XX.XX"}`

## Проверка DNS на роутере

### Проверка служб:
```
netstat -tanp | grep LISTEN
```
Ожидается: sing-box на `127.0.0.42:53` и `127.0.0.1:1602`, dnsmasq на `127.0.0.1:53` и `192.168.1.1:53`

### Проверка dnsmasq:
```
uci show dhcp.@dnsmasq[0]
```
Должно быть: `noresolv='1'` и `server='127.0.0.42'`

## Проверка правил nftables
```
nft list ruleset | grep -B3 mark
```
Счетчики пакетов для `198.18.0.0/15` должны быть ненулевыми.

## Проверка VPN-интерфейса
```
curl -v --interface wg0 ifconfig.me
```
