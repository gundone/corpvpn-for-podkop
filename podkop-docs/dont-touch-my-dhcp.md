# Don't Touch My DHCP -- Podkop

Источник: https://podkop.net/docs/dont-touch-my-dhcp/

По умолчанию podkop редактирует `/etc/config/dhcp`, чтобы прописать sing-box в качестве основного DNS-сервера. Это необходимо для работы FakeIP/FakeDNS. Эта опция отключает данное поведение, позволяя вам вручную настраивать dnsmasq.

В первую очередь необходимо для использования опции `server`.

## Важно

> DNS-сервер `127.0.0.42` должен быть единственным сервером в dhcp.

Когда эта опция включена, podkop оставляет `/etc/config/dhcp` без изменений. Вы должны вручную настроить:

```
option noresolv '1'
option cachesize '0'
list server '127.0.0.42'
```

Без этих настроек podkop не может функционировать.

## Конфигурация DNS для конкретных доменов

Для маршрутизации определенных доменов через конкретные DNS-серверы:

```
option noresolv '1'
option cachesize '0'
list server '/itdog.info/1.1.1.1'
list server '127.0.0.42'
```

Эта конфигурация маршрутизирует itdog.info и все поддомены через 1.1.1.1.
